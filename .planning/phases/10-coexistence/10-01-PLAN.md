<feature>
  <name>Planning Directory Parser</name>
  <files>internal/compat/compat.go, internal/compat/compat_test.go</files>
  <behavior>
    ## Types

    ProjectState struct:
    - CurrentPhase int (0 if not found)
    - CurrentPlan int (0 if not found)
    - TotalPlans int (0 if not found)
    - LastActivity string (empty if not found)
    - Progress string (e.g. "87%" — empty if not found)

    PhaseEntry struct:
    - Number int
    - Name string
    - Goal string (empty if not found in details)
    - Complete bool (from [x] checkbox)
    - Plans int (plan count, 0 if not found)

    FallbackStatus struct:
    - ProjectName string
    - CoreValue string
    - State ProjectState
    - Phases []PhaseEntry

    ## ParseState(content string) ProjectState

    - Test 1: Parse "Phase: 9 of 10" -> CurrentPhase=9
    - Test 2: Parse "Plan: 2 of 2 in current phase" -> CurrentPlan=2, TotalPlans=2
    - Test 3: Parse "Progress: [xxx] 87%" -> Progress="87%"
    - Test 4: Parse "Last activity: 2026-03-21 -- blah" -> LastActivity contains "2026-03-21"
    - Test 5: Empty string -> zero-value ProjectState (no panic)

    ## ParseRoadmap(content string) []PhaseEntry

    - Test 1: Parse "- [x] **Phase 1: Binary Scaffold**" -> PhaseEntry{Number:1, Name:"Binary Scaffold", Complete:true}
    - Test 2: Parse "- [ ] **Phase 10: Coexistence**" -> PhaseEntry{Number:10, Name:"Coexistence", Complete:false}
    - Test 3: Parse full ROADMAP.md content -> 10 phases, first 9 complete
    - Test 4: Parse "**Goal**: A single Go binary..." -> Goal populated on correct phase
    - Test 5: Empty string -> empty slice (no panic)

    ## ParseProject(content string) (name string, coreValue string)

    - Test 1: Parse "# gsd-wired" -> name="gsd-wired"
    - Test 2: Parse "GSD's full development lifecycle..." from Core Value section -> coreValue populated
    - Test 3: Empty string -> empty strings (no panic)

    ## DetectPlanning(dir string) bool

    - Returns true if dir/.planning/ directory exists (os.Stat)
    - Returns false otherwise

    ## BuildFallbackStatus(dir string) (FallbackStatus, error)

    - Reads dir/.planning/STATE.md, ROADMAP.md, PROJECT.md
    - Calls ParseState, ParseRoadmap, ParseProject
    - Returns combined FallbackStatus
    - Missing files are non-fatal (partial result)
  </behavior>
  <implementation>
    Use regexp for field extraction per Claude's Discretion. Compile patterns at package level (like reqLabelPattern in Phase 2).

    Key patterns to extract from STATE.md:
    - `Phase: (\d+) of (\d+)` for current phase
    - `Plan: (\d+) of (\d+)` for plan progress
    - `Progress: \[.*?\] (\d+%)` for progress percentage
    - `Last activity: (.+)` for last activity

    Key patterns for ROADMAP.md:
    - `- \[(x| )\] \*\*Phase (\d+): (.+?)\*\*` for phase list
    - `\*\*Goal\*\*: (.+)` under Phase Details sections

    Key patterns for PROJECT.md:
    - First `# (.+)` line for project name
    - Content after "## Core Value" heading for core value

    All parsers: pure functions (string in, struct out). No file I/O in parse functions. BuildFallbackStatus handles file reads. DetectPlanning does the os.Stat check.

    Per D-09: NO write operations to .planning/ anywhere in this package. Read-only functions only.
  </implementation>
</feature>
