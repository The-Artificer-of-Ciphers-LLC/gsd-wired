  <action>
    Create internal/mcp/init_project.go containing:

    1. An initProjectArgs struct with JSON tags matching the schema from 05-RESEARCH.md:
       - project_name (string, required)
       - what (string, required)
       - why (string, required)
       - who (string, optional)
       - done_criteria (string, required)
       - tech_stack (string, optional)
       - constraints (string, optional)
       - risks (string, optional)
       - mode (string, required: "full"|"quick"|"pr")
       - pr_url (string, optional for pr mode)

    2. An initProjectResult struct: { project_bead_id string, context_bead_ids []string, files_written []string }

    3. func handleInitProject(ctx, state, args initProjectArgs) (*mcpsdk.CallToolResult, error):
       a. state.init(ctx) — triggers bd init if .beads/ absent (INIT-05)
       b. Create project epic bead via state.client.CreatePhase(ctx, 0, args.ProjectName, args.What, args.DoneCriteria, []string{"gsd:project"}) — phase 0 = project level
       c. Create 4 context child beads via state.client.CreatePlan for each category that has non-empty content:
          - "decisions" with context=args.Constraints (if non-empty)
          - "tech-stack" with context=args.TechStack (if non-empty)
          - "risks" with context=args.Risks (if non-empty)
          - "done-criteria" with context=args.DoneCriteria
          Per D-06 (Claude's discretion): use category beads, not per-answer beads. 4 categories max.
       d. Write PROJECT.md to state.beadsDir using a Go template (per D-07: human-readable, parallel to beads):
          ```
          # {project_name}

          ## What
          {what}

          ## Why
          {why}

          ## Who
          {who}

          ## Done Criteria
          {done_criteria}

          ## Tech Stack
          {tech_stack}

          ## Constraints
          {constraints}

          ## Risks
          {risks}

          ---
          *Initialized: {timestamp}*
          *Mode: {mode}*
          ```
          Skip empty sections for quick mode.
       e. Create .gsdw/ directory if absent, write .gsdw/config.json:
          ```json
          {"project_name": "...", "initialized": "...", "mode": "..."}
          ```
       f. Return initProjectResult with bead IDs and file paths.

    4. Update registerTools in tools.go: add init_project tool registration with JSON schema from research. The handler unmarshals args into initProjectArgs and calls handleInitProject.

    5. Update server.go: change slog.Debug tools count from 8 to 10.

    6. Update TestToolsRegistered in tools_test.go: expect 10 tools, add "init_project" and "get_status" to wantNames.

    IMPORTANT: The CreatePhase call for the project epic uses phaseNum=0 as a convention for "project-level" (not a real phase). The CreatePlan calls for context beads use planID like "init-decisions", "init-tech", etc. Parent is the project epic bead ID.

    IMPORTANT: For quick mode (D-05), only project_name, what, why, done_criteria are populated. Skip sections with empty values in PROJECT.md.
  </action>

    1. A statusResult struct:
       ```go
       type statusResult struct {
           ProjectName  string      `json:"project_name"`
           CurrentPhase *phaseInfo  `json:"current_phase"`
           ReadyTasks   []taskInfo  `json:"ready_tasks"`
           TotalPhases  int         `json:"total_phases"`
           OpenPhases   int         `json:"open_phases"`
       }
       type phaseInfo struct {
           BeadID   string `json:"bead_id"`
           Title    string `json:"title"`
           PhaseNum int    `json:"phase_num"`
           Status   string `json:"status"`
           Goal     string `json:"goal"`
       }
       type taskInfo struct {
           BeadID string `json:"bead_id"`
           Title  string `json:"title"`
           PlanID string `json:"plan_id"`
       }
       ```

    2. func handleGetStatus(ctx, state) (*mcpsdk.CallToolResult, error):
       a. state.init(ctx)
       b. Query project bead: state.client.QueryByLabel(ctx, "gsd:project") — find the project epic (the one created by init_project with phaseNum=0). Extract project_name from Title.

       d. Query ready tasks: state.client.ListReady(ctx) — map to taskInfo array.
       e. Return statusResult. Per D-09: dashboard format. Per D-10: GSD-familiar terms (phases, plans, waves). Never expose bead IDs to the user in the SKILL.md rendering — but the JSON response includes them for programmatic use.

    3. Update registerTools in tools.go: add get_status tool registration with empty input schema (no args needed). Handler calls handleGetStatus.
