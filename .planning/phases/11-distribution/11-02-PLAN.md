  <name>Task 1: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
    Create `.github/workflows/release.yml` with this exact content per D-06, D-08, D-15, D-19, and research STACK.md CI/CD section:

    ```yaml
    name: Release

    on:
      push:
        tags: ['v*']

    permissions:
      contents: write
      packages: write

    jobs:
      release:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
            with:
              fetch-depth: 0

          - uses: actions/setup-go@v5
            with:
              go-version: '1.26'

          - name: Run tests
            run: go test ./... -v

          - uses: docker/login-action@v3
            with:
              registry: ghcr.io
              username: ${{ github.actor }}
              password: ${{ secrets.GITHUB_TOKEN }}

          - uses: goreleaser/goreleaser-action@v6
            with:
              version: v2.14.3
              args: release --clean
            env:
              GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
              HOMEBREW_TAP_GITHUB_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
    ```

    Key points:
    - Pin GoReleaser to v2.14.3 per D-15 (do NOT use `latest`)
    - `fetch-depth: 0` required for GoReleaser changelog generation
    - `permissions: packages: write` required for ghcr.io push
    - `permissions: contents: write` required for GitHub Releases
    - `HOMEBREW_TAP_TOKEN` is a repository secret (PAT with contents:write on the homebrew-gsdw repo per D-19)
    - Tests run before release to catch any issues
    - This workflow handles Linux binaries and container images only
    - macOS signed binaries come from the local GoReleaser run on the developer's Mac per D-08

    Also create the `.github/workflows/` directory if it does not exist.
