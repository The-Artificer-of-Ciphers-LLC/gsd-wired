```go
type verifyPhaseArgs struct {
    PhaseNum   int    `json:"phase_num"`
    ProjectDir string `json:"project_dir"`
}

type criterionResult struct {
    Criterion string `json:"criterion"`
    Passed    bool   `json:"passed"`
    Method    string `json:"method"`   // "file_exists", "go_test", "grep", "manual"
    Detail    string `json:"detail"`
}

type verifyPhaseResult struct {
    PhaseNum int               `json:"phase_num"`
    Passed   bool              `json:"passed"`
    Results  []criterionResult `json:"results"`
    Failed   []string          `json:"failed"`
}
```

Handler logic:
- state.init(ctx)
- QueryByLabel("gsd:phase"), find epic matching phase_num (same pattern as execute_wave)
- Parse AcceptanceCriteria: split by newline, strip leading "N. " or "- " prefixes, skip empty lines
- For each criterion, call checkCriterion(ctx, criterion, projectDir):
  - extractFilePath: look for known file extensions (.go, .ts, .md, .json, .yaml, .toml) or path separators (/) in criterion text. If found, use os.Stat.
  - If criterion contains "test" (case-insensitive): run `exec.CommandContext` with 60-second timeout for `go test ./...` in projectDir. Use ctx from MCP handler (Pitfall 5).
  - If criterion contains a function/type name pattern (uppercase identifier): use grep-like file scan. For v1, mark as "manual" if no clear file path.
  - Default: method="manual", passed=false (conservative).
- Aggregate: Passed = all criteria passed. Failed = criterion text for failed items.
- verify_phase does NOT call create_plan_beads (per D-10, SKILL.md handles remediation).
