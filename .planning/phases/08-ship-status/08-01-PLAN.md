    `handleCreatePrSummary(ctx, state, args)`:
    1. Call `state.init(ctx)`
    2. Query `gsd:project` label for project name (graceful fallback to "")
    3. Query `gsd:phase` label for all phases
    4. Find the target phase by matching `phaseNumFromMeta(bead.Metadata) == args.PhaseNum`
    5. If not found, return `toolError`
    6. Build PR title: `"Phase {N}: {phase_title}"` — short, GSD-familiar per D-05
    7. Build PR body as markdown:
       - `## Summary` section with phase goal (from bead.Description)
       - `## Requirements` section listing requirement IDs from the phase bead's labels (filter labels matching `reqLabelPattern` from the mcp package — it's already compiled at package level in execute_wave.go or similar; if not found, use a local `regexp.MustCompile("^[A-Z]+-\\d+$")`)

       - Footer: `Generated by gsd-wired`
    8. Build branch name: `"ship/phase-{N}"` (lowercase, simple)
    9. Return `prSummaryResult{Title, Body, BranchName}`

    **advance_phase.go** (per D-06, D-07):

    Create `advancePhaseArgs` struct with `PhaseNum int`, `Reason string`. Create `advancePhaseResult` struct with `Closed *graph.Bead`, `Unblocked []graph.Bead`, `NextPhase *phaseInfo` (nullable).

    `handleAdvancePhase(ctx, state, args)`:
    1. Call `state.init(ctx)`
    2. Query `gsd:phase` label for all phases
    3. Find the target phase epic bead by `phaseNumFromMeta` matching
    4. If not found, return `toolError("no phase epic found for phase N")`
    5. Call `state.client.ClosePlan(ctx, bead.ID, args.Reason)` — reuse existing close pattern per D-06
    6. After closing, find the next open phase (lowest phase number with status "open" and phaseNum > args.PhaseNum) from the already-queried phases list. Construct a `phaseInfo` for it if found.
    7. Return `advancePhaseResult{Closed, Unblocked, NextPhase}`
