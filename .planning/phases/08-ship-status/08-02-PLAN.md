    Content structure — the ship flow per D-03, D-06, D-07:

    **Your role:** You are a ship coordinator. Create PRs and advance project state. Never mention beads or graph structures to the developer. Use phase numbers and plan IDs.

    **Step 1: Determine phase** — From $ARGUMENTS or call `get_status` for current phase.

    **Step 2: Get PR summary** — Call `create_pr_summary` with `phase_num`. Receive `{ title, body, branch_name }`.

    **Step 3: Display PR preview** — Show the developer the PR title and body in a formatted block. Display: "Creating PR... (auto-continuing in 30 seconds...)" per D-02.

    **Step 4: Create PR** — Per D-03, use `gh pr create`:
    ```bash
    git checkout -b {branch_name} 2>/dev/null || git checkout {branch_name}
    git push -u origin {branch_name}
    gh pr create --title "{title}" --body "{body}"
    ```
    If `gh` fails (not installed or not authenticated), display the error and instruct the developer to install/authenticate `gh` CLI, then retry.

    **Step 5: Display PR link** — Show the PR URL returned by `gh pr create`.

    **Step 6: Advance phase** — Call `advance_phase` with `phase_num` and `reason="Shipped as PR"`. Display:
    ```
    Phase {N} complete.
    ```

    **Step 7: Show next phase** — If `advance_phase` returns a `next_phase`, display:
    ```
    Next up: Phase {next_phase.phase_num}: {next_phase.title}
    Ready to plan? (auto-continuing to /gsd-wired:plan {next_phase.phase_num} in 30 seconds...)
    ```
    Wait 30 seconds, then auto-proceed to `/gsd-wired:plan {next_phase.phase_num}` per D-07.

    If no next phase, display:
    ```
    All phases complete. Project shipped.
    ```

    **Error handling:**
    - If `create_pr_summary` returns an error, display it and stop
    - If `gh pr create` fails, display error with instructions to fix, then retry
    - If `advance_phase` fails, display error but note the PR was already created
    - If no changes to ship (no commits ahead of main), display: "No changes to ship. All work is already on main." and skip PR creation, proceed directly to phase advancement.
