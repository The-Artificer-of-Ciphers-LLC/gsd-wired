## What Was Built

### internal/container/runtime.go

Runtime interface with three implementations:

- **DockerRuntime**: `docker run -d --name gsdw-dolt -p 127.0.0.1:3307:3306 -e DOLT_ROOT_HOST=% -v <dir>:/var/lib/dolt dolthub/dolt-sql-server:latest`
- **PodmanRuntime**: Same as Docker but with `0.0.0.0:3307:3306` binding (required for Podman on macOS)
- **AppleContainerRuntime**: `container run -d --name gsdw-dolt -p 127.0.0.1:3307:3306 -v <dir>:/var/lib/dolt dolthub/dolt-sql-server:latest`

DetectRuntime priority: Apple Container (macOS 26 + arm64) → Docker → Podman. All detection injected via DetectOpts for hermetic tests.

### internal/container/compose.go

WriteComposeFragment writes `gsdw.compose.yaml` to any directory. Features:
- Services: gsdw-dolt with healthcheck (mysqladmin ping), restart unless-stopped, port 127.0.0.1:3307:3306
- Overwrite protection: returns clear error with `--force` guidance when file exists
- Header comment: "Generated by: gsdw container start"
- text/template for maintainability

### internal/deps/check.go

Extended checkContainerRuntime to check `container` binary first (priority order: container → docker → podman). Added:
- `isMacOS26OrNewer()` helper with `GSDW_MOCK_MACOS_MAJOR` env injection for tests
- `installHelp["container"]` entry pointing to macOS 26 Tahoe + Apple Silicon requirement
