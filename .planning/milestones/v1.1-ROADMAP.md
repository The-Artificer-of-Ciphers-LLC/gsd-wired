# Milestone v1.1: Installation Toolkit

**Status:** SHIPPED 2026-03-22
**Phases:** 11-14
**Total Plans:** 8

## Overview

Distribution, setup UX, container management, and connectivity configuration. Makes gsd-wired installable and configurable by any developer through standard channels.

## Phases

### Phase 11: Distribution Infrastructure
**Goal**: gsd-wired is installable by any developer through standard channels without manual build steps
**Depends on**: Nothing (pipeline work, no source changes)
**Requirements**: DIST-01, DIST-02, DIST-03, DIST-04, DIST-05, DIST-06
**Plans**: 2 plans

Plans:
- [x] 11-01-PLAN.md — Version ldflags injection, --json flag, GoReleaser config, Dockerfile
- [x] 11-02-PLAN.md — GitHub Actions release workflow (verified end-to-end with v0.0.1-rc7)

**Success Criteria**:
1. brew tap + brew install works
2. go install works
3. Tag push triggers release pipeline
4. macOS binary passes Gatekeeper
5. docker pull from ghcr.io works

### Phase 12: Setup UX
**Goal**: A developer who just installed gsdw can verify their environment and get guided to a working state in one command
**Depends on**: Phase 11
**Requirements**: SETUP-01, SETUP-02, SETUP-03, SETUP-04, SETUP-05
**Plans**: 2 plans

Plans:
- [x] 12-01-PLAN.md — Dependency detection package (internal/deps), check-deps and doctor CLI commands
- [x] 12-02-PLAN.md — Interactive setup wizard with install method selection

### Phase 13: Container Support
**Goal**: A developer can start a Dolt server in a container with a single command using whichever runtime is available
**Depends on**: Phase 12
**Requirements**: CNTR-01, CNTR-02, CNTR-03, CNTR-04, CNTR-05, CNTR-06, CNTR-07
**Plans**: 2 plans

Plans:
- [x] 13-01-PLAN.md — Runtime abstraction (Docker/Podman/Apple Container), DetectRuntime, compose fragment
- [x] 13-02-PLAN.md — CLI subcommands (gsdw container start/stop with pre-flight checks)

### Phase 14: Connectivity
**Goal**: gsdw knows how to reach the Dolt server and automatically passes configuration to every bd command
**Depends on**: Phase 13
**Requirements**: CONN-01, CONN-02, CONN-03, CONN-04, CONN-05, CONN-06
**Plans**: 2 plans

Plans:
- [x] 14-01-PLAN.md — Connection config package, health check, env var injection into client.go
- [x] 14-02-PLAN.md — Connect wizard, doctor Connection section, root wiring

## Milestone Summary

**Key Decisions:**
- GoReleaser v2.14.3 pinned in CI (never use latest)
- Apple Container gated on macOS 26 + Apple Silicon
- beads bug #2073: BEADS_DOLT_SERVER_HOST/PORT env var injection
- FlexPort type for JSON port parsing robustness
- GPG signing with --pinentry-mode loopback for CI
- Docker manifest uses arch-suffixed tags to avoid conflicts

**Issues Resolved:**
- TestInitCmdWritesFiles: graceful skip when bd not on PATH
- GoReleaser postinstall field removed (unsupported)
- Docker manifest tag collision (amd64/arm64 suffix pattern)
- GPG signing in headless CI (loopback pinentry + passphrase flag)

**Technical Debt Incurred:**
- TestPostToolUseBeadUpdate flaky (pre-existing, timing-sensitive)
- macOS notarization not tested in CI (requires local goreleaser run)
